かつて私は、構造を設計し、自らコードを打ち込むという直接的な行為にプログラマーとしての「craftsmanship」を感じていた。しかしAIの発展により、私はコードを書く代わりにAIへ指示を出し、出力を選ぶ存在になりつつある。

これはcraftsmanshipの所在を探す実験の一時的な断面である。

本作は完成したプログラムではない。人間とAIが往復する「意思決定の軌跡」のログを独自のルールで可視化している。

画面上部から中盤には、AIへ指示を出し、試行錯誤を重ねた複雑な軌跡が記録されている。しかし画面下部に向かうにつれ、線の密度は下がり、一定のリズムへと変化する。制作の最終盤で、AIが与えられたルールを処理しきれずに意図した出力が得られなくなり、システムを完成させるために、私自身の手で直接コードを記述する道を選んだためだ。

AIを導くプロセスに新たなcraftsmanshipは宿るのか、それとも人間の直接的な介入にこそ残存しているのか。
そして、AIがさらに進化し、私たちが手を入れる余白すら失われたとき、そのcraftsmanshipは消滅するのか、あるいは別の形へと変容を遂げるのだろうか。

この軌跡を前に、私自身もまだ明確な答えを出せずにいる。本作は、AI時代における「つくること」への私自身の戸惑いの記録であり、これを見た方と共に、これからのcraftsmanshipについて考えるための見取り図である。

**【Rule / 凡例】**
制作プロセスのログデータは、以下の規則に従い幾何学模様へと変換される。

- **進行方向**：左上を起点とし、左右へ蛇行しながら下段へ進む時間軸。
- **点と線**：コードの変化量。変化が多いほど点が増え、近接する二点が結ばれる。
- **斜線の密度**：コードの入力・生成量。（※AIによる生成と人間による入力は区別して描画）
- **白抜き（空白）**：コードの削除量。修正や破棄の痕跡。
- **強調された枠線**：AIへのプロンプト指示（介入）を行ったタイミング。
